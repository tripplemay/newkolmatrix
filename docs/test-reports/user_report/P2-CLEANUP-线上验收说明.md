## ② F002 深色模式持久化

**修复前的症状：** 切到深色后一刷新就退回浅色，等于白切。

**路径：** 任意 admin 页，如 https://newkol.guangai.ai/admin/today → 顶栏右侧的**月亮 / 太阳图标**

| # | 操作 | 期望 | 你的结论 |
|---|---|---|---|
| 2.1 | 点图标切到深色 | 立即变深色 | |
| 2.2 | **刷新页面** | **仍是深色** ← 修复前这里是坏的 | |
| 2.3 | 刷新的**那一瞬间**盯着看 | **不应**先闪一下白底再变深（这是内联脚本的价值所在） | |
| 2.4 | 切回浅色 → 刷新 | 仍是浅色 | |
| 2.5 | 深色态下切到别的页面（创作者库 / 项目 / 洞察） | 深色跟着走，不会莫名变回浅色 | |
| 2.6 | （可选）F12 → Application → Local Storage → 删掉 `kolmatrix.colorMode` → 刷新 | 回落**浅色**（默认态） | |

### ⚠️ 这条会顺带放大一个既有问题，请你留意

深色模式以前是「一刷新就没」的临时状态，所以**深色下的样式瑕疵极少被看到**。现在它变持久了，你会**长时间待在深色里**，那些瑕疵会集中暴露出来。

**本批只做了「持久化」，没有做深色样式打磨。** 所以：

- 如果你发现某些页面深色下难看 / 对比度不够 / 某块还是白的 —— 那**多半是既有状态，不是这次改坏的**
- 但**请照样记下来**（写在下面「深色巡视」表里）。这正是本批最有价值的副产品：把一个以前看不见的问题变得看得见了
