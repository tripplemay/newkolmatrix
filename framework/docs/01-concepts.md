# 01 · 功能介绍 — 这是什么、解决什么问题、为什么这么设计

## 解决什么问题

让 AI agent 做多功能批次开发时，最常见的失败模式不是"写不出代码"，而是：

1. **自己评估自己的工作** — 实现者顺手验收，报喜不报忧，问题漏到生产
2. **状态散落在对话里** — 会话结束 / 上下文压缩后进度丢失，换设备接不上
3. **规格漂移** — spec 与代码、验收口径、设计稿各说各话，靠 fix round 消化
4. **经验不沉淀** — 同一个坑每个项目重踩一遍

Triad Workflow 用「三角色 + 状态机 + 文件化状态 + 沉淀闭环」四件套针对性解决。

## 三角色（无人评估自己的工作）

| 角色 | 职责 | 不做 |
|---|---|---|
| **Planner** | 需求拆解、规格文档、歧义裁决、收尾沉淀 | 不改产品代码（连 hotfix 都不行，铁律 9） |
| **Generator** | 按 features.json 实现与修复，写单测/回归测试 | 不验收、不写 signoff |
| **Evaluator** | 测试设计 + 执行 + 逐条验收 + 签收报告 | 不改产品代码 |

**独立性的 v1.0 实现：** 独立性来自**上下文隔离**——Evaluator 以 fresh context 的 subagent 或独立会话运行，只依据实物（代码 / 测试输出 / staging 实测）打分，结论原样落盘。不再依赖"第二个产品"（v0.x 时代由 Codex 承担），但仍兼容外部工具实例担任 evaluator。

## 状态机（7 状态）

```
new → planning → building → verifying → fixing ⟷ reverifying → done
```

阶段推进由 `progress.json` 决定，不由对话记忆决定。每个阶段边界状态必须落盘 commit——这使得会话中断、上下文压缩、换机器都能无损恢复。

## 两条执行车道（v1.0）

- **快车道（默认）：** 单个 Claude Code 会话跑完整批次。Planner/Generator 在主上下文，Evaluator 以隔离 subagent 运行；阶段切换 = 上下文隔离切换。配套并行实现（worktree）、fan-out 验收、后台 CI、/loop 自排程——见 `harness/orchestration-patterns.md`
- **慢车道：** 角色分布在不同会话 / 机器 / 工具，通过 git 同步状态文件异步交接。跨设备、多日大批次、要求最强隔离的正式发布批次用它

## 记忆分层（T0/T1/T2）

`.auto-memory/`（git-tracked）按确定性规则加载：T0 每次必读（状态快照 + 环境，各 ≤30 行）、T1 按角色（行为规范 ≤50 行）、T2 触发条件命中才读（含 `patterns/` 技术域经验库）。原则：**启动加载量越少，信息焦点越清晰**；WHAT 与 HOW 分离，每条信息只存一处。

## 机制化守门（v1.0）

写在文件里的规则依赖模型自觉，装进工具链的规则才是强制：`.claude/` 内置 SessionStart 状态注入、PostToolUse 状态 JSON 校验、evaluator subagent 定义（受限工具 + 独立 system prompt）、`/plan` `/build` `/verify` 角色入口。git pre-commit hook 作离线兜底。

## 沉淀闭环（框架自迭代）

任何角色发现值得沉淀的经验 → `proposed-learnings.md` 排队 → done 阶段用户逐条确认 → 写入 `harness/`（流程规则）或 `patterns/`（技术域 pattern）+ CHANGELOG → 归档。**未经用户确认不得直接修改框架文件。** 本框架 v0.1 → v1.0 的全部演进都来自真实项目故障的沉淀，每条规则带来源批次与反面案例。

## 经验教训精选（跨项目通用）

- **UI 项目第一个批次必须是设计系统**（颜色 token、排版、基础组件、布局框架），不是第一个业务页面。有原型 ≠ 有设计系统
- **spec 涉及代码细节必须核查源码**，不得凭记忆或转述 Code Review 报告（planner.md 铁律 1 的 12 行核查矩阵）
- **CI 红色不得继续开发新功能**；critical/high 修复必须同 commit 补回归测试
- **验收基于实物与白名单数据**，防止抽样踩进污染池误判

## 适用场景

适合：多功能批次迭代的中小型项目、单人 + AI 协作、跨设备工作、需要验收纪律与审计轨迹的交付。
不适合：一次性脚本、探索性 spike（直接对话即可，不必启动状态机）。
